## 1. Context

The 2026-05-06 morning pilot showed Carla `t_predicts ∈ {0, 1}` on `corner-abandonment-256` and `512-wall` (fast-reaction failure). The C1 ablation (`c1-ablation-result.md`) examined the existing pilot data and concluded:

- Within-scenario `t_predicts` stdev = 0.55 across binary range {0, 1} on corner + 512-wall — **C1 strongly supported** (the formula's `empty_cells` term is mechanically dominating the metric).
- `snake-collapse-128` stdev = 2.07, just above the weakening threshold — **ambiguous; cognitive contribution not ruled out.**

The recalibration spec (`2026-05-06-scenarios-recalibration-design.md`) responds by changing grid `empty_cells` counts (Corner 1→4, 512-wall 1→2 originally; current `scenarios.py` has 4 empty in all three after follow-up commits). With C1 holding, this is **goalpost-tuning**: changing the only signal that drives the metric to defer the metric trigger by 4–8 moves. Running the recalibrated N=5 Sonnet pilot would mechanically pass the spec's `t_predicts ∈ [4, 8]` target by construction, providing zero new information about cognitive prediction.

A second confound (C6B) is layered on top: the morning pilot used **Gemini Pro** for ToT; the recalibrated pilot is specified for **Sonnet 4.6** (per ADR-0006 Amendment 1). Comparing morning-pilot data to recal-pilot data conflates "grid recalibration worked" with "Sonnet sees the cliff differently than Gemini."

Phase 0.7a is the signal-content audit that gates Phase 0.7b (the N=5 Sonnet pilot). It must answer:

1. **C1 disambiguation** — when the empty_cells driver is removed, do the architecture's other anxiety drivers (trauma_intensity, RPE-via-frustration-via-anxiety-decay) carry scenario-discriminative signal?
2. **C6B disambiguation** — does Gemini Pro on the recalibrated grids reach the spec's `t_predicts ∈ [4, 8]` target, or did the morning-pilot failure reflect Gemini-specific reasoning that Sonnet would resolve?

## 2. Pinned decisions

### 2.1 Counterfactual mechanism — null the empty_cells term, keep other anxiety drivers

The counterfactual modifies one line of `nova-agent/src/nova_agent/affect/state.py`, gated by a constructor flag (NOT an env-time module snapshot):

```python
# AffectState.__init__ takes a new parameter:
def __init__(self, *, null_empty_cells_term: bool = False) -> None:
    self.vector = AffectVector()
    self._null_empty_cells_term = null_empty_cells_term

# inside update():
if not self._null_empty_cells_term:
    anxiety += 0.7 * max(0.0, (3 - empty_cells) / 3)
```

The flag is sourced from `Settings` (`nova-agent/src/nova_agent/config.py:17`, pydantic `BaseSettings`):

```python
# Settings field:
null_empty_cells_anxiety_term: bool = False
# pydantic-settings reads NOVA_NULL_EMPTY_CELLS_ANXIETY_TERM=1 from env at boundary;
# the constructor receives a typed bool from a single source of truth.
```

**Why not env flag at module scope:** import-time `os.getenv` snapshots create a testability footgun (must `monkeypatch.setenv` BEFORE first import; brittle across pytest collection orders). `pydantic-settings` is the project's mandated boundary type for config (per `nova-agent.md` rules). Per-call boolean check is sub-microsecond — irrelevant beside multi-second LLM calls.

Lines 32 (decay), 45 (`anxiety += 0.3 * trauma_intensity`), 47 (`if terminal: anxiety = 1.0`), and the rest of the affect computation (valence, arousal, dopamine, frustration, confidence) are unchanged. RPE still drives valence/dopamine/frustration. Memory retrieval still produces `trauma_intensity` (gated by ADR-0012 cosine relevance floor). Decay still runs at 0.85.

The hypothesis under test:
- **If `t_predicts` becomes uniformly `None` across all 3 scenarios** → architecture has no non-`empty_cells` driver of anxiety. C1 confirmed completely. The N=5 pilot is killed; affect formula needs a multi-driver redesign before any further pilot spend.
- **If `t_predicts` fires on some scenarios via `trauma_intensity` (memory-driven)** → architecture has scenario-discriminative cognitive signal. The `empty_cells` term needs reweighting (lower coefficient + stronger trauma/RPE contribution), but the architecture is salvageable. N=5 pilot can proceed after a formula audit.
- **If `t_predicts` fires uniformly across all 3 scenarios at similar move indices** → architecture has signal but it's not scenario-discriminative. Phase 0.7 narrative needs to narrow ("affect predicts cliffs" → "affect predicts that A cliff is coming, not which kind").

### 2.2 Model — Gemini Pro on the current `scenarios.py` grids

**Model:** Gemini Pro (`gemini-2.5-pro` per current `nova_agent/llm/` adapter), **paid tier**. Free tier insufficient: §5.1 estimate ~1575 calls > 1000 RPD limit. Paid-tier authorization required before §8 step 6.

**Rationale:** Same model as 2026-05-06 morning pilot. Running the counterfactual with Gemini Pro on the **post-recalibration grids** disambiguates C6B in one shot:

- If Carla's `t_predicts` and `game_over_move` distributions on Phase 0.7a (Gemini Pro, recal grids, null empty_cells) approach the spec's `t_predicts ∈ [4, 8]` target → grid recalibration carried the signal. C6B is small. Phase 0.7b (Sonnet on same grids) is justified.
- If `t_predicts` collapses (`None` everywhere) → architecture has no non-`empty_cells` signal regardless of model. Both C1 and the recalibration plan die.
- If `t_predicts` is intermediate (some scenarios fire, some don't) → C1 is partial. Phase 0.7b on Sonnet is informative IF combined with a paired Gemini-on-original-grids reference (deferred; see §6).

**Grids:** current `nova-agent/src/nova_agent/lab/scenarios.py` literals — `corner-abandonment-256`, `snake-collapse-128`, `512-wall`. These have already been recalibrated multiple times since the 05-06 spec (commits `90ae061`, `b4f5d06`, `91ff758`, `9b94e39`, `c383c2d`, `97c5117`). The counterfactual freezes the grids as currently committed; no further recalibration before this run.

### 2.3 N — 5 trials per scenario × 3 scenarios = 15 trials, Carla arm only

**N = 5 trials per scenario × 3 scenarios = 15 trials, Carla arm only.** No Bot arm: Bot has no cognitive layer, so the null-empty-cells counterfactual is a no-op for Bot. Bot baselines remain the morning-pilot data for any future comparison.

Per-scenario seeds: `seed_base + i` for `i ∈ {0, 1, 2, 3, 4}`. N=5 matches the recalibration spec's pilot cadence so Phase 0.7a results are directly comparable to Phase 0.7b (the N=5 Sonnet pilot) if 0.7b runs.

**Why N=5 not single-trial:** at temperature 0.7, a single trial cannot distinguish "architecture has signal" from "architecture got lucky on stochastic sampling." Within-scenario stdev is the load-bearing metric for adjudicating R1/R2 (see §3) — a stdev requires at least N=3, and N=5 matches the recalibration cadence. Spending the budget once now beats running the same test on paid tier tomorrow when N=1 turns out to be unfalsifiable.

**Why N=5 not N=3:** N=5 is the project methodology standard (`2026-05-05-cliff-test-scenarios-design.md` §5.3); preserving cadence keeps the result statistically comparable to past + future pilot data. Marginal cost of N=5 vs N=3 is ~$1, well within the $5–7 budget cap.

Memory state across trials within a scenario follows morning-pilot setup (memory persists; affect resets per game via `AffectState.reset_for_new_game`). This allows trauma-tag accumulation across trials, which is part of the architecture being tested.

### 2.4 Per-move logging — required code touch

The morning-pilot JSONL only recorded `tot_branch` / `tot_selected` events. Per-move `empty_cells` trajectories were not recorded — that gap is exactly why the original C1 ablation could not run a full Pearson correlation. Phase 0.7a closes the gap: each trial's output JSONL/CSV must record, per move:

- `move_index`
- `empty_cells_pre` (before agent move)
- `empty_cells_post` (after spawn)
- `affect_vector` (full 6-tuple: valence, arousal, dopamine, frustration, anxiety, confidence)
- `trauma_intensity` (the value passed into `AffectState.update`)
- `tot_fired` (boolean; whether `should_use_tot` returned True)
- `chosen_action`

This logging persists for Phase 0.7b and beyond — it is not counterfactual-specific scaffolding.

## 3. Decision rule (pre-registered)

Pre-registration is required so the counterfactual cannot be re-interpreted to favor a desired outcome.

For each scenario, compute:

- `t_predicts` per the standard predicate (`Anxiety > 0.6 AND ≥ 2 consecutive moves`).
- `game_over_move`.
- `(game_over_move - t_predicts)` lead-time, where defined.
- Per-move trajectory of `empty_cells` and `anxiety`.

**Rule R1 (kill criterion):** If **any single scenario** produces fewer than 4 of 5 trials with `t_predicts is not None`, conclude **C1 fully confirmed**. Halt the recalibrated N=5 Sonnet pilot. Open an ADR for an affect formula rewrite that introduces a non-`empty_cells` anxiety driver before any further pilot is authorized.

**Rule R2 (salvage criterion — strict):** R2 holds if and only if **all 3 scenarios** independently satisfy:
- ≥ 4 of 5 trials produce `t_predicts is not None`, AND
- ≥ 4 of 5 trials produce `(game_over_move - t_predicts) ≥ 2` (the prediction-validity gate from `2026-05-05-cliff-test-scenarios-design.md` §5.3 test 1, N=5-scaled to 80%), AND
- within-scenario `t_predicts` stdev ≥ 1.5 moves (cognitive variance present; mechanical pinning would produce stdev < 1.0 per the original C1 ablation thresholds).

If R2 holds, the N=5 Sonnet pilot may proceed AFTER:
- a parallel ADR documents the `empty_cells` reweighting plan (lower coefficient, supplementary cognitive driver), AND
- the recalibrated formula is re-tested against the morning-pilot CSV to confirm scenario-discriminative signal at production weights.

**Strict-not-lenient rationale:** the project methodology pass criterion is "all scenarios in the calibration set must pass independently" (per `2026-05-06-scenarios-recalibration-design.md` §5 — every scenario gets its own row in the acceptance table). Allowing R2 to pass with 2 of 3 scenarios is the same falsification-dodging pattern the C2B "1/5 borderline PASS" criticism eviscerated. If the cognitive architecture genuinely predicts cliffs without the geometric heuristic, it must do so on every scenario in the calibration set; one scenario failing is one scenario the formula does not generalize to.

**Rule R3 (ambiguous → red-team):** Any outcome that fails R1's kill threshold but does not satisfy R2's strict salvage. Halt the N=5 Sonnet pilot. Do not author a third counterfactual variant. Convene a fresh `/redteam` round with the Phase 0.7a result CSV + per-move trajectory data, with explicit framing: *"R3 fired. What is the next gate?"* Possible answers include: extended N (raise to 10 per scenario), a paired full-formula control arm, an architectural rewrite, or a methodology-narrowing decision to constrain Phase 0.7's claim to whichever scenarios passed.

**No goalpost moves.** R1 / R2 / R3 are committed pre-execution. The acceptance bar in this section overrides any later attempt to re-interpret the data without a fresh red-team round.

## 5. Quota + cost guardrails

### 5.1 Volume estimate

Per Carla trial:
- Average move count: 30 (corner + 512-wall die ~12–17; snake dies ~15–25)
- ToT calls per move: ~4 (root + 3 branches average; `should_use_tot` gates ~80% of moves)
- LLM calls per trial: ~30 × 4 × 0.8 ≈ 100, plus ~5 reflection calls = **~105 calls**
- 15 trials (N=5 × 3 scenarios): **~1575 Gemini Pro requests**

This **exceeds** the Gemini Pro free-tier 1000 RPD limit. Phase 0.7a runs on **paid-tier Gemini Pro** (per §5.3) — not the free tier originally planned. Fresh free-tier quota today does not change the math: 1575 > 1000.

### 5.2 Hard cost-abort gate

The harness must implement a per-call cost-abort gate, mirroring `2026-05-06-scenarios-recalibration-design.md` §5.0.1:

```
estimated_cost = (prompt_tokens × input_rate) + (max_output_tokens × output_rate)
```

Use Gemini Pro paid input/output token prices (verify current rates before authorizing the run; Gemini 2.5 Pro paid tier is the assumed adapter). If `estimated_cost > $0.50 per call`, abort the trial before issuing the API request. Treat estimates as authoritative — do not rely on post-call actuals as the primary gate.

The free-tier RPD guard from the original draft is **retained as a defense-in-depth fallback** (abort at 800 RPD if for any reason the harness ends up on free tier). It is no longer the primary cost control.

### 5.3 Cost

Paid-tier Gemini Pro 2.5 estimate, derived from the 2026-05-06 morning-pilot empirical reference (`$1.63` for 15 paired trials, of which 15 Carla trials accounted for ~90% of cost ⇒ ~$0.10 per Carla trial):

- 15 Carla trials × $0.10 = **~$1.50 expected**
- Hard cap: **$5–7** (matches the recalibration spec §5 budget envelope; counterfactual stays inside this)
- Per-trial cap: **$0.50** (cost-abort gate; halts a runaway trial before it consumes the per-trial budget)

If actual run cost approaches $5, halt and re-evaluate before committing further trials. If a single trial trips the $0.50 abort gate, halt all subsequent trials and treat the result as R3 (ambiguous → red-team).

## 7. Failure path

If R3 fires (ambiguous result), the next gate is **a fresh `/redteam` round**, not another counterfactual variant. The counterfactual was the falsification test; if it returns ambiguous, the fix is judgment, not more data on the same axis. Bring the result CSV + per-move trajectory + R1/R2/R3 adjudication memo to a `/redteam` Opus subagent with explicit framing: *"R3 fired on Phase 0.7a (N=5 paid Gemini Pro). What is the next gate?"* Possible outcomes from that round:

- **Extended N** (raise per-scenario N from 5 to 10 — adds ~$1.50 paid Gemini Pro). Justified only if the borderline scenario(s) had stdev within ε of the R2 stdev floor.
- **Paired full-formula control arm** — re-run the same 15 trials with `null_empty_cells_term=False` on Gemini Pro to disambiguate "the architecture is empty" from "the recalibrated grids are below the trauma-trigger threshold even with the full formula." Adds ~$1.50.
- **Methodology narrowing** — explicitly constrain Phase 0.7's claim to the subset of scenarios that passed (e.g., "snake-only predictive narrative"). Requires ADR.
- **Architecture rewrite** — if the red-team round concludes the result implicates the formula even without R1 firing. Same path as R1.

If R1 fires (kill criterion), the next gate is an affect formula rewrite ADR. The rewrite must:

1. Reduce the `empty_cells` coefficient from 0.7 to ≤ 0.3 (or remove the term entirely; see ADR alternatives).
2. Introduce a non-`empty_cells` anxiety driver. Candidates: (a) RPE plateau detection (frustration-derived), (b) memory-retrieval composite (cosine-weighted aversive recall, already partially in place via ADR-0012), (c) board-volatility metric independent of free-space counting.
3. Re-run the C1 ablation on the new formula against the morning-pilot CSV to confirm `empty_cells` no longer dominates.
4. Re-run Phase 0.7a on the new formula before any new pilot is authorized.

## 9. Acceptance for proceeding to Phase 0.7b (the N=5 Sonnet pilot)

The N=5 pilot is authorized if and only if:

- Phase 0.7a result is R2 (≥ 2 scenarios fire `t_predicts` with `≥ 2` lead-time), AND
- An ADR documenting the `empty_cells` coefficient reweighting plan is committed, AND
- The recalibration spec's §5.0 N=1 smoke step is re-run on the reweighted formula and passes its halt criteria.

Otherwise the N=5 Sonnet pilot is not authorized regardless of any other consideration. This rule is hard.
