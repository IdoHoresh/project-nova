## Context Checkpoint — 2026-05-09 00:05 UTC

**Last shipped:** `ef4989b` — `feat(llm): per-call cost-abort gate for paid Gemini Pro runs`

**Architectural intent:**
Phase 0.7a §8 step 5 — defense-in-depth on the paid-tier LLM seam. The
existing M-07 `SessionBudget` cap stops cumulative spend (slow-burn
guard); this new per-call gate stops a single runaway request before
the API call is issued (oversized `max_tokens`, huge system prompt,
accidental loop). Two complementary controls, not one extended.
Threshold default $0.50/call mirrors recalibration spec §5.0.1 verbatim
so the same operational mental model carries over from Phase 0.7's
Sonnet-tier pilot to Phase 0.7a's paid Gemini Pro pilot. Gate wired
into BOTH `cliff_test.main()` (active 0.7a path) AND
`trauma_ablation._build_llms()` (preventative coverage; not a 0.7a
path but ships now to avoid an incomplete-rollout footgun the security
reviewer flagged).

**Edge-case warnings:**
- **Char-count // 4 heuristic UNDER-counts tokens vs real tokenizers in
  both directions** (English ~3.5 chars/token actual; CJK ~1
  char/token actual). The estimate generally under-states true cost.
  Threshold is set conservatively low precisely so a small
  under-estimate cannot mask a runaway. Phase 0.8 may swap for a
  tokenizer-backed estimate if CJK or under-counting paths emerge in
  production. The first code-reviewer pass flagged my docstring
  saying the *opposite* — caught and fixed pre-commit.
- **Per-call gate must fire BEFORE `pre_charge`** so the session
  budget is not consumed by a rejected call. Test
  `test_per_call_cap_aborts_when_estimate_exceeds_cap` asserts
  `budget.spent == 0.0` after the abort — load-bearing invariant.
- **`per_call_cap_usd=0.0` disables the gate** (matches
  `BudgetGuard`/`SessionBudget` "0 = no limit" convention). Easy
  footgun if someone sets `NOVA_PER_CALL_COST_ABORT_USD=0` to silence
  a false alarm; runaway protection vanishes silently. `None` also
  disables — that path is for backward compatibility with positional
  `BudgetedLLM(llm, budget)` callers.
- **structlog audit log emits model + max_tokens + estimate + cap on
  gate trip** but NEVER `system` / `messages` content (security.md
  crown-jewels). If extending the log fields, keep prompt content out.
- **Settings field NOT bootstrapped via `get_settings()` in
  `cliff_test.main()`** — env read directly via `os.environ.get` to
  decouple from secrets validation. `trauma_ablation._build_llms`
  uses `s.per_call_cost_abort_usd` because it already has the
  Settings instance loaded. Two paths, same env var, same default.
- **Backward-compatible kwarg-only param** preserves all existing
  `BudgetedLLM(llm, budget)` positional callers (test fixtures,
  `test_decision_baseline.py:406`). Default `None` skips the gate.

**Rejected alternatives:**
- **Tokenizer-backed pre-call estimate** (tiktoken / sentencepiece) —
  rejected; per-provider tokenizer dependency adds install weight, and
  the conservative-low threshold + post-call true-up makes the
  heuristic acceptable for the Phase 0.7a abort-gate use case.
- **Per-call cap as an extension of `SessionBudget` itself** —
  rejected; would conflate cumulative-cap and single-call-cap concerns
  in one class. Kept the per-call gate inside `BudgetedLLM` so
  `SessionBudget` stays single-purpose.
- **Module-scope env read in `protocol.py`** — rejected; runtime config
  must flow through pydantic-settings or explicit kwargs.
  `cliff_test.main()` does the env read so the protocol layer stays
  pure-config.
- **Wire only `cliff_test`, defer `trauma_ablation`** — rejected after
  security-reviewer flagged incomplete rollout. 5 lines added to
  `trauma_ablation._build_llms` for parity; no behavior change unless
  envvar is set.
- **Inline `0.50` literal** — rejected; Settings field
  `per_call_cost_abort_usd` lets paid-tier runs override per-spec
  §5.2 without code changes, and the env-var alias matches the rest
  of the project's NOVA_* convention.
- **Skip structlog log on gate trip (rely on BudgetExceeded
  message)** — rejected after code-reviewer flagged spec §5.2
  "treat estimates as authoritative — do not rely on post-call
  actuals." Audit trail of which calls aborted is operationally
  necessary for post-pilot diagnosis.
- **Test the structlog emit** — deferred; would couple test to log
  internals. Existing assertions on `inner.call_count == 0` +
  `budget.spent == 0.0` verify behavior; log emission is observability,
  not contract.
