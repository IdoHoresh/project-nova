**Architectural intent:**
M-07's `pre_charge()` pattern is intentional — it increments `spent` *before* yielding to the event loop so concurrent asyncio coroutines in `asyncio.gather(carla_trial, bot_trial)` cannot both pass the cap check on the same stale `spent` value. The `BudgetedLLM` wrapper lives in `protocol.py` alongside the `LLM` Protocol and `PRICING` to keep the gate visible to any future LLM implementer. The `BudgetExceeded` re-raise fix (commit 6437c11) was non-obvious: the broad `except Exception` retry loop in `baseline._call_with_api_retry` was silently swallowing the exception, making the gate a no-op for the bot arm only.
