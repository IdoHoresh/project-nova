# Phase 0.7b Harness Decision — trauma_intensity Treatment

**Date:** 2026-05-09
**Triggered by:** Methodology-code audit §2.1 ([`2026-05-09-methodology-code-audit-results.md`](2026-05-09-methodology-code-audit-results.md)) — production-vs-lab harness gap. Audit found `cliff_test.py:444` hardcodes `trauma_intensity_passed = 0.0`, nulling one of three anxiety drivers during Phase 0.7a. Audit framed this as a methodological side-question for Phase 0.7b plan, not part of ADR-0013 anxiety-formula scope.
**Decision route:** /redteam (Opus subagent) on the original Option C recommendation; red team challenge accepted; Option B locked.

---

## 1. The question

For the Phase 0.7b counterfactual re-run (falsification gate for the anxiety-formula rewrite), should the harness:

- **A.** Switch from `cliff_test.py` to the production main-loop path (`main.py`), with retrieval enabled and trauma_intensity computed from real aversive recall.
- **B.** Keep `cliff_test.py` as-is, including the `trauma_intensity_passed = 0.0` hardcode at line 444 and the disabled retrieval path. ← **Decision.**
- **C.** Extend `cliff_test.py` to enable retrieval and compute real `trauma_intensity` (mirroring `lab/trauma_ablation.py:394-411`), preserving the deterministic harness shell.

---

## 2. Decision

**Option B.** Phase 0.7b runs against `cliff_test.py` unchanged from Phase 0.7a. `trauma_intensity = 0.0` for all events. Retrieval remains disabled.

The independent variable between Phase 0.7a and Phase 0.7b is **only the anxiety-formula rewrite** (frustration → anxiety wiring + K-tuning + 3rd driver per ADR-0013 scope). Trauma's contribution is tested separately in Phase 0.8 against `lab/trauma_ablation.py`, which is the harness designed for that question.

---

## 3. Reasoning

### 3.1 Single-variable experimental design

Phase 0.7a: anxiety formula A, trauma=0. Phase 0.7b: anxiety formula B, trauma=0. The only variable that changes is the formula. Pass/fail attribution is clean:

- **Pass under B:** the rewrite alone closed the predictive-lead-time gap. No confound.
- **Fail under B:** the rewrite failed under conditions identical to Phase 0.7a. Clean kill verdict, no need to re-run to disambiguate.

Option C would change two variables simultaneously (formula + retrieval-enable). Pass/fail attribution becomes ambiguous: "did the rewrite work, or did re-enabling trauma rescue it, or did both contribute?"

### 3.2 Memory-seed problem makes Option C effectively equivalent to B

`cliff_test.py:398-401` spins a fresh `tempfile.TemporaryDirectory` per trial — every cliff trial starts with empty memory. Cliff scenarios are short and do not include game-over events, so `tag_aversive` (which requires a game-over hook per `main.py:97`) never fires during a trial. With empty memory + no aversive-tag events, retrieval has nothing to surface. `trauma_intensity` would compute to 0.0 anyway.

Re-enabling retrieval in `cliff_test.py` without scenario redesign therefore yields the same `trauma_intensity = 0.0` output as Option B, at the cost of additional code, additional non-determinism sources (embedding latency, top-k tie-breaking, vector store insertion-order), and methodological scope creep. The `trauma_ablation.py` precedent does NOT transfer cleanly: trauma_ablation builds episodic memory across episodes; cliff_test does not.

### 3.3 Phase 0.8 is the trauma-question phase

`docs/superpowers/specs/2026-05-07-phase-0.8-trauma-ablation-design.md` is the pre-registered harness for testing trauma's contribution. `lab/trauma_ablation.py:394-411` already enables retrieval and computes real `trauma_intensity`. Running Phase 0.8 on the rewritten architecture, as already planned, is the correct way to test trauma's contribution. Pre-empting that test in Phase 0.7b would duplicate infrastructure and split the trauma-testing surface across two runners.

### 3.4 Audit verdict re-read

Audit §2.1 names the trade-off but does not mandate the fix. Line 117 explicitly: *"The §2.1 production-vs-harness gap is a methodological side-question for the Phase 0.7b re-run plan, not part of the ADR's anxiety-formula scope."* The audit's "falsification-dodge by another name" framing flags that the trade-off must be **owned**, not that it must be **reversed**. Owning the trade-off means: declare it explicitly, justify it, document where the trauma question is answered (Phase 0.8). This decision memo discharges that obligation.

## 4. Costs of the decision

- **Adjudication memo amendment:** the Phase 0.7a adjudication memo (`docs/superpowers/plans/2026-05-09-phase-0.7a-counterfactual-impl.md` and any follow-on `docs/external-review/phase-0.7a-result.md` artifacts) should note that the "trauma=0 across 658 events" observation was a harness artifact, not a code gap. **Action item, not blocking this decision.**
- **ADR-0013 preamble (when drafted):** ADR-0013 must include a one-paragraph note in its preamble (or "Methodology" / "Falsification gate" section) preempting the reviewer question "why didn't you re-enable trauma in 0.7b?" Wording guidance:

  > Phase 0.7b runs under the same harness as Phase 0.7a (`cliff_test.py` with `trauma_intensity = 0.0`) to preserve a single-variable comparison: the rewrite is the only change between runs. Trauma's contribution to predictive lead-time is tested separately in Phase 0.8 against `lab/trauma_ablation.py`, which seeds episodic memory across episodes and is the harness designed for that question.

- **No code changes.** `cliff_test.py:444` stays as-is.

---

## 5. Alternatives considered (rejected)

### Option A — production main-loop path

Rejected: requires live emulator, OCR noise, large harness rewrite, non-determinism. Scope creep beyond what the audit finding justifies.

### Option C — extend `cliff_test.py` with retrieval

Rejected for four reasons:

1. Confounds the experimental design (rewrite + retrieval-enable change simultaneously).
2. Memory-seed problem: fresh tempdir + no game-over events means trauma can't fire even with retrieval enabled. Effectively equivalent to B with extra wiring.
3. Re-introduces the network/DB latency confound the original `=0.0` pin was designed to remove (`cliff_test.py:439-440` comment).
4. Duplicates infrastructure that already exists in `lab/trauma_ablation.py`. §4.4 scope-escalation pattern.

The original Option C recommendation was a hybrid framed as "address audit + preserve determinism + smallest diff," and the red team correctly identified it as motivated-reasoning-prone synthesis. See LESSONS.md entry on hybrid recommendations.

---

## 6. Pre-registration symmetry note

Trauma is **not** a candidate driver in the anxiety-formula rewrite (ADR-0013). The audit lists ToT-branch-disagreement as the candidate third driver (`audit-results.md` §2.6, citing `2026-05-09-phase-0.7a-counterfactual-design.md` §7 R1 candidate (c)). Because trauma is not in scope for the rewrite, the harness condition for trauma does not need to be pre-registered in ADR-0013 — it correctly lives in this Phase 0.7b plan memo.

Had trauma been a candidate driver in the rewrite, this decision would have belonged in ADR-0013 itself rather than here.
