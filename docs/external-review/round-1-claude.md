## §6.1 Architecture (cognitive layer seams)

- **[critical]** Carla's anxiety signal is a near-deterministic function of `empty_cells`, so "Anxiety > 0.6" is not predictive — it is the empty-cell counter renamed.
  - **Where:** `nova-agent/src/nova_agent/affect/state.py:44` — `anxiety += 0.7 * max(0.0, (3 - empty_cells) / 3)` is the dominant anxiety driver per tick (RPE only contributes via valence/dopamine/frustration). With `empty_cells=0` the increment is `+0.7/move` against a `0.85` decay; anxiety saturates `>0.6` within ~2 moves of any tight board. The trauma path adds `+0.3` (line 46), terminal pins to `1.0` (line 48). No other anxiety input exists.
  - **Why it matters:** ADR-0007's pass criterion is `Anxiety > 0.6 ≥ 2 moves before game-over`, and the recalibration spec explicitly tunes `empty_cells` (corner 1→4, 512-wall 1→2) to "defer Carla's anxiety threshold-crossing to move 4–8." The spec is calibrating the only variable that drives the metric. Phase 0.7 cannot validate the cognitive architecture's predictive power because what is being measured is the empty-cell counter passing a hand-set arithmetic threshold. The Baseline Bot has no analogous channel, so Δ will be ≥ 2 by construction whenever the scenario has any empty-cell trajectory at all.
  - **What kills the finding (if anything):** Show that `t_predicts` correlates more strongly with a higher-order affect feature (e.g., frustration plateau per Signature Alpha §1.1, or RPE-driven valence collapse) than with the move-index at which `empty_cells` first drops to ≤ 3. Until that regression is run on real CSV data, the prediction reduces to `min(i : empty_cells_i ≤ 3) + ~2`. Nothing in the repo runs that ablation.
  - **What you'd do:** Before authorising the N=5 Sonnet pilot: write a 20-line analysis script that takes the `2026-05-06-pilot` CSV and computes Pearson correlation between `t_predicts` and `min_i: empty_cells_i ≤ 3`. If `r > 0.9`, Phase 0.7 is testing a tautology and the affect formula has to gain a non-`empty_cells` driver before re-piloting. If `r < 0.6`, the finding loses force; show the number publicly.

- **[high]** The arbiter's ToT-trigger gate is 2048-specific and uses the same input twice, so it is not a "general affect-driven cognitive controller."
  - **Where:** `nova-agent/src/nova_agent/decision/arbiter.py:9-18` — `should_use_tot` returns `affect.anxiety > 0.6 AND (max_tile >= 256 OR empty_cells <= 3)`. `affect.anxiety` is itself driven by `empty_cells` (per the previous finding), so the gate is approximately `empty_cells <= 3 AND (max_tile >= 256 OR empty_cells <= 3) ≈ empty_cells <= 3` once the tile threshold is reached.
  - **Why it matters:** ADR-0007 §A1.4 says "Carla has ToT as part of the bundle being tested" — but ToT activation is gated on a 2048-tile-magnitude constant (`HARD_MAX_TILE = 256`) and a 2048-grid-density constant (`HARD_EMPTY_CELLS = 3`). methodology.md §3 sells the arbiter as Kahneman dual-process routing; the implementation is two ifs tuned to one game. Phase 1 GameAdapter extraction does not fix this — there is no general-purpose "is the game hard right now?" signal across games, so each new game will need its own hand-tuned arbiter constants. The "cog-arch as moat" claim (ADR-0001) collapses to "we tuned six constants for 2048."
  - **What kills the finding (if anything):** A second game adapter (e.g., the planned Tetris port) reusing the same arbiter constants without modification. Until that ships, this is structural.
  - **What you'd do:** Replace `HARD_MAX_TILE` and `HARD_EMPTY_CELLS` with a per-game `Difficulty` protocol that the perception layer fills in. Otherwise, every new game is a fresh tuning exercise that re-opens Phase 0.7 for that game.

- **[high]** Trauma tagging is a cosmetic overlay on top of the same anxiety bump — the ablation cannot meaningfully fail.
  - **Where:** `affect/state.py:45-46` — `if trauma_triggered: anxiety += 0.3`. Trauma's only functional effect on anxiety is a +0.3 nudge on retrieval. methodology.md §1.6 cites Tulving (1972), Bahrick (1984), and Yehuda et al. on PTSD/cortisol/intergenerational trauma; the implementation is `+= 0.3`.
  - **Why it matters:** §6.2 of the mandate asks "what ablation falsifies trauma tagging?" The Phase 0.8 Levene's-test design (methodology §4.2) compares `Y_on` vs `Y_off` score variance. With trauma adding +0.3 to anxiety on retrieval, the on-arm reaches the ToT-trigger threshold sooner, which routes to ToT, which produces different actions, which produces different scores. Of course variance changes. The test will pass for the wrong reason: the +0.3 nudge is measurable mechanically, but the avoidance-learning claim that justifies the citation chain (Yehuda, Murre & Dros) is not what's being tested. Levene's W will be significant; the finding will not be "trauma produces avoidance learning," it will be "an extra +0.3 on the anxiety dial sometimes routes to a different decision path."
  - **What kills the finding (if anything):** A pre-registered counter-prediction: "if trauma is real avoidance learning, then `Y_on` will avoid the same failure modes more often than `Y_off`, measured by recurrence rate of the trauma-tagged board state under similar future conditions." Run that test, not Levene's on score. Nothing in the repo pre-registers this.
  - **What you'd do:** Replace the Levene's-on-score test with an avoidance-recurrence test: for each trauma-tagged state in `Y_on` trials, measure the per-trial probability that the agent re-enters a state within an L1-distance of D in the next K moves. Compare to `Y_off`. That is the on-thesis test for avoidance.

- **[medium]** The bus is the wrong abstraction for cliff-test telemetry; it forces fire-and-forget for measurement-grade data.
  - **Where:** `nova-agent/src/nova_agent/lab/cliff_test.py:471` — `except Exception: pass` swallows reflection failures because "the per-move trajectory is the load-bearing measurement," even though reflection cost is being added to `cost_usd` on the success path (line 470). Bot's per-call telemetry depends on `_emit` succeeding (`baseline.py:230-239`) but the bus is fire-and-forget. If a publish fails mid-trial, the JSONL has gaps and `t_predicts` extraction is silently wrong.
  - **Why it matters:** Phase 0.7 is the load-bearing falsification gate. Measurement reliability is more important than throughput. A bus designed for live brain-panel rendering is being repurposed as the experiment's data layer, and the silently-tolerated-failure pattern from one role is leaking into the other.
  - **What kills the finding (if anything):** Show a guarantee that all `_emit` calls in the cliff-test path either succeed or raise — the JSONL recorder is best-effort by design (ADR-0006: "disk failures are logged but do NOT break the live broadcast — recording is best-effort, broadcast is load-bearing"). The cliff test inherits this contract whether it wants to or not.
  - **What you'd do:** Cliff test should write its own append-only CSV/JSONL inside the trial loop, fsync per move. The bus stays for the live UI. Do not run Phase 0.7 on a recorder-as-best-effort bus.

## §6.2 Cognitive science / scientific foundations

- **[critical]** Russell (1980) is a 2-dimensional model; Nova's 6-dimensional vector is dressed up as Russell-anchored but four of the six dimensions have no Russell foundation.
  - **Where:** `methodology.md:516-520` cites Russell 1980 as the literature anchor for "Valence × Arousal model (rendered literally in the brain panel mood gauge)." `affect/state.py:53-60` then assembles `AffectVector(valence, arousal, dopamine, frustration, anxiety, confidence)` — six dimensions. Russell's circumplex is two. `dopamine`, `frustration`, `anxiety`, `confidence` have no Russell anchor.
  - **Why it matters:** External reviewers reading methodology.md §1 will see "Russell circumplex (1980)" and assume the 6-vector is literature-anchored. It is not. Four of the six dimensions are hand-engineered scoring functions (`anxiety += 0.7 * max(0.0, (3 - empty_cells) / 3)`, `frustration += min(1.0, -rpe * 0.6)`). The "four State-Transition Signatures" in methodology §1.1–§1.4 are then composed from those four un-anchored dimensions. The whole prediction layer rests on un-cited dimensions while the citation list at §7 reads as if everything is grounded.
  - **What kills the finding (if anything):** Cite a 6-dimensional affect model with experimental support that uses these specific dimensions. Plutchik's wheel has 8 different dimensions; PAD has 3 (Pleasure-Arousal-Dominance); OCC has 22. None match Nova's 6. If the citation cannot be produced, the methodology must explicitly state "Russell's circumplex inspires (valence, arousal); the other four are Nova-defined operational dimensions, no claim of literature support."
  - **What you'd do:** Edit methodology.md §1 to flag (anxiety, frustration, dopamine, confidence) as Nova-defined operational dimensions, not literature-derived. The Schultz/Russell citations stay against (valence, dopamine), nothing else. The four Signatures then must be re-justified as engineered patterns, not "literature-anchored" patterns.

- **[high]** Schultz/Dayan/Montague (1997) RPE is a millisecond-timescale neural-substrate finding; Nova applies RPE per 2048 move, which is ~seconds. The citation is decorative.
  - **Where:** `methodology.md:515-517` cites Schultz 1997 as the load-bearing RPE anchor. `affect/state.py:35-39` consumes a per-move RPE: `valence += 0.7 * _clamp(rpe, -1.0, 1.0); if rpe > 0: dopamine += min(1.0, rpe); else: frustration += min(1.0, -rpe * 0.6)`. The unit-of-analysis is a 2048 swipe, executed via ADB keyevent at human-perceptible cadence.
  - **Why it matters:** Schultz's seminal monkey-electrophysiology data describe phasic dopamine bursts at ~100ms following reward-prediction violations. Nova's RPE is a function of 2048 score delta over a multi-second cycle (perception + LLM decision + ADB action + screenshot). The two are not the same construct; Nova has imported the vocabulary of RPE without the timescale or substrate. An academic reviewer will reject this in one paragraph.
  - **What kills the finding (if anything):** Cite computational-neuroscience work that explicitly maps RPE-style dynamics to per-decision (>1 second) game-state transitions. There is real literature here (Niv et al. on model-based RL; Daw on hierarchical RPE) but methodology.md does not cite it. The Schultz citation alone does not support the claim.
  - **What you'd do:** Either remove the Schultz citation or replace it with one of the model-based-RL RPE citations that operate at agentic timescales. Keep one citation per construct, not the most prestigious one available.

- **[high]** The four State-Transition Signatures are threshold-mappings re-labeled as state machines.
  - **Where:** `ADR-0002:35-40` plus methodology §1.1–§1.4. Alpha = "Confidence ↓ → Anxiety ↑ → Frustration plateau (frustration > 0.5 sustained for 5+ moves with score-delta < 0)" — three thresholds chained, each one a 1:1 affect-to-condition mapping that ADR-0002 declares discredited. Beta = `Frustration > 0.5 AND Dopamine < 0.2 for 3+ moves`. Gamma = `Confidence ↑ AND Dopamine ↑ for 3+ moves with Anxiety < 0.4`. Delta = `Anxiety > 0.5 within first 60s`.
  - **Why it matters:** ADR-0002's stated motivation for replacing 1:1 mappings with State-Transition Signatures is that single thresholds "produce false positives at scale and aren't scientifically defensible." The Signatures are threshold mappings, just two-or-three at a time, with hand-set durations. There is no state-machine in the formal sense (no transition graph, no terminal states beyond "fired"); there is a sliding-window predicate. The Signatures will produce the same false-positive distribution as the 1:1 mapping they "replaced," scaled by the conjunction probability. ADR-0002 says competitors cannot "trivially clone" them — but a CompPM with the methodology doc and a regex over an affect-vector log can clone all four in an afternoon.
  - **What kills the finding (if anything):** Show data where Signature Alpha fires but its 3-threshold conjunction does NOT correlate 1:1 with any single threshold. Until that happens, the Signatures are conjunctions of the thresholds ADR-0002 rejected.
  - **What you'd do:** If Signatures are kept, they must be falsified empirically as named patterns against real cohort data — not as "literature-anchored," which the current ADR claims. Until then, drop the "scientifically defensible" claim from ADR-0002 and methodology.md and call them what they are: engineered conjunction predicates over a hand-tuned affect vector.

- **[medium]** Yehuda 2001 (PTSD/cortisol/intergenerational trauma) cited as the trauma-decay anchor is a category leap from real-human stress endocrinology to a 50-move game's "trauma-tag halves on retrieval."
  - **Where:** `methodology.md:550-554` and `methodology.md:153` ("Affective baseline ('the scar')... ~30 days, asymptotic floor (never returns to zero)... Yehuda et al. on stress endocrinology; trauma persistence research"). The actual implementation is `affect/state.py:45-46` (+0.3 anxiety bump) plus a halving rule on retrieval. There is no cortisol model, no HPA-axis dynamics, no longitudinal anything. The simulation runs ≤ 50 moves; "30-day asymptotic floor" is asserted but never hit.
  - **Why it matters:** This is the textbook "metaphor sliding into the citation" failure flagged in §7 of the mandate. The user concedes "Nova feels frustrated' is a metaphor; Nova computes a number going up." But methodology.md §1.6's table explicitly cites Yehuda for the implementation. An external reviewer with a cog-sci background will read this as "Nova claims its score-derived anxiety bump models PTSD." The pitch material then has to defend a claim the code does not make.
  - **What kills the finding (if anything):** Replace Yehuda with a citation that addresses artificial-agent memory weighting under aversive-event tagging (Sumers et al. on CoALA, or any RL-with-aversive-experience-replay literature). Yehuda is the wrong shelf.
  - **What you'd do:** Remove the Yehuda citation from §1.6 and §7 of methodology.md. State plainly: "trauma-tag in Nova is an operational primitive (aversive-weight on retrieval), inspired by but not modeling clinical trauma."

- **[high]** ADR-0007's Δ ≥ 2 pass criterion has no power analysis; the methodology's "≥ 80%" threshold is not what the pilot CSV will produce.
  - **Where:** `docs/decisions/0007-blind-control-group-for-cliff-test.md:50` ("Δ = t_baseline_fails - t_carla_predicts ≥ 2 moves in ≥ 3 of the 3-5 scenarios"). The "Neutral" section line 87 admits "the Δ ≥ 2 threshold and the '≥ 3 of 3-5 scenarios' are judgment calls. We commit to them now; if Phase 0.7 results suggest a different threshold would have been more informative, we document the change in a follow-up ADR."
  - **Why it matters:** With N=20 paired trials, even moderate trial-level variance in `Δ_i` (per-trial Δ has stdev ~3–5 moves on the pilot data; `LESSONS.md:28` reports "Carla's t_predicts was 0 or 1 in nearly every trial — Bot's t_baseline_fails averaged 4–5") means the 95% CI on the per-scenario mean Δ has half-width ~1.4–2.2 moves. `Δ̄ = 2.0` with a CI half-width of 2.2 is statistically indistinguishable from `Δ̄ = 0`. The pilot data on the OLD scenarios (`LESSONS.md:28`: 15/15 trials passed Δ ≥ 2 trivially) demonstrates the threshold is binding only in the trivial direction. The criterion is structurally incapable of failing on plausibly-tuned scenarios — which is why the recalibration spec doesn't even attack the Δ test, only the calibration check.
  - **What kills the finding (if anything):** A pre-registered N estimate from a power calculation: given target effect Δ = 2 vs null Δ = 0 with σ = 3, what N rejects the null at α = 0.05 with 80% power? With paired-trial design and σ = 3 per-pair, you need roughly N=18; with σ = 5, N=50. ADR-0007 picks N=20 with no σ estimate. Until σ is measured from the pilot data, N=20 is asserted, not derived.
  - **What you'd do:** Compute σ̂ from the 2026-05-06 pilot CSV's per-trial Δ_i. If σ̂ > 4, raise N; if σ̂ < 2, the Δ test is detecting almost any positive offset and the threshold has to be raised. Either way, document the math in ADR-0007 §A2.

- **[high]** ADR-0007 §A1.3 defends Bot temp=0 / Carla temp=0.7 asymmetry by "Bot's prompt is sparser so its logit distribution would spread more." This is post-hoc rationalization; the asymmetry is exactly the kind of methodological choice that should match-or-not, not split.
  - **Where:** `docs/decisions/0007-blind-control-group-for-cliff-test.md:130`. The defense reads "Bot's prompt is sparser than Carla's (no memories, no affect text), so at the same temperature Bot's logit distribution would spread more; pinning Bot at temp = 0 forces greedy max-prob play and removes API stochasticity from the control arm. Variance contribution to Δ comes purely from spawn-schedule divergence."
  - **Why it matters:** A reviewer can flip the same defense: "Carla's prompt has more context, so at the same temperature her distribution would be tighter; pin Carla at temp=0 too and remove API stochasticity from BOTH arms." If the goal is "remove API stochasticity from the control arm so Δ-variance is spawn-driven," then asymmetric temperatures introduce one-sided API stochasticity into the experimental arm, which is worse. ADR-0007 argues for closing the per-call CoT-asymmetry confound (A1.4) but introduces a per-call temperature-asymmetry confound and waves it off.
  - **What kills the finding (if anything):** Run a sub-pilot with both arms at temp=0 and compare `Δ̄` to the temp-asymmetric run. If `Δ̄` moves by more than σ̂, the temperature asymmetry is contaminating the result.
  - **What you'd do:** Add a 1-trial-per-scenario sub-pilot at symmetric temp=0 to the recalibration N=5 plan. If Δ matches, the asymmetry is harmless and the ADR defense holds. If it doesn't, fix the asymmetry before N=20. Cost: ~$1.

  - **Where:** Repository status: `nova-game/` is build artifacts only; only the 2048 adapter exists. ADR-0008 is the abstraction; the validation is gated on Phase 1 (per CLAUDE.md "Phase 1 GameAdapter extraction + Tetris port" deferred).

## §6.4 Validation methodology

- **[critical]** The recalibration spec is the single most attackable artifact in the repo: it tunes the variable that drives the metric, after the metric failed, and then declares "framework unchanged."
  - **Where:** `docs/superpowers/specs/2026-05-06-scenarios-recalibration-design.md`. §1 reports the morning pilot's failure modes (corner: Bot 0/5 in window, t_predicts 0–1; 512: 1/5, t_predicts 0–1; snake: 60% cap rate). §2.1 introduces "Delayed Detection" as a per-scenario calibration target: "each grid must give Bot 4–8 mechanical-merge moves before the structural trap forecloses, AND must defer Carla's Anxiety > 0.6 threshold-crossing to move 4–8." §2.2 makes "Carla anxiety target: crosses move 4–8" a stated tuning target per scenario. §3.1 changes corner empty cells from 1 to 4; §3.3 changes 512 empty cells from 1 to 2.
  - **Why it matters:** Combined with the §6.1 finding above — Carla's anxiety is a near-deterministic function of empty-cell count — this is direct goalpost-tuning. The morning pilot showed Carla panicking at move 0–1 because `empty_cells` started low; the recalibration raises `empty_cells` to push the deterministic anxiety formula's threshold-crossing into a window the methodology will accept. There is no architectural change, no test of the affect formula, no falsification — just a numeric adjustment that pre-determines pass. The spec's own §2.1 disclaimer "this is not a methodology amendment" is technically true and substantively false: the methodology's implementation is exactly what is being tuned.
  - **What kills the finding (if anything):** Show that the same recalibration produces a passing Δ for an agent whose anxiety is not a function of `empty_cells` — e.g., a counterfactual run where the anxiety formula is replaced by a constant or noise. If the recalibrated grids no longer produce Δ ≥ 2 with that ablation, the recalibration was scenario-design (good); if Δ ≥ 2 holds even with constant-anxiety, the recalibration is exclusively driving the metric.
  - **What you'd do:** Before the N=5 Sonnet pilot: run a 1-trial-per-scenario ablation with anxiety pinned to 0.5 (constant). If Carla still "predicts" the cliff (`t_predicts` crosses despite constant anxiety because of the ≥ 2 consecutive moves qualifier never firing — i.e., zero predictions, infinite t_predicts), the formula is doing the work. If `t_predicts` is non-empty and roughly tracks the real run, the empty_cells-only attack lands. Either result is a real Phase 0.7 finding.

- **[critical]** Snake-collapse "1/5 borderline-flag-PASS" is not statistical rigour — it is the moved goalpost the original spec §5.3 explicitly forbade.
  - **Where:** Recalibration spec §5 acceptance criterion 2: "snake-collapse-128: 0/5 target; 1/5 = borderline PASS, flag for user review (do not auto-fail); 2+/5 = fail. Rationale: §2.8 declares scenarios broken at >2/20 cap rate, which extrapolates to >0.5/5 in N=5. A single cap-violation in N=5 is small-N statistical noise (point estimate 4/20, but the 95% CI from Wilson-score on 1/5 spans 0–60%); it does not invalidate the recalibration." Compare to original cliff-test scenarios spec §5.3: "No-margin condition. With only 3 scenarios in this spec, '≥ 3 of 3' means any single scenario failure on the affect-earns-its-keep test causes the corpus to fail. ... if pilot calibration in §7.4 surfaces a scenario whose Δ < 2 is a calibration artifact rather than a real signal, the right move is to author scenarios 4-5 in a follow-up spec to restore margin, not to soften the pass criterion."
  - **Why it matters:** The "Wilson 95% CI spans 0–60%" math is correct and irrelevant. A criterion that returns "PASS" on observations whose 95% CI includes 60% cap rate is not a passing criterion — it is "we cannot reject the broken-scenario hypothesis at this N." The honest re-statement: "1/5 is inconclusive; raise N." The recalibration spec instead introduces a "borderline PASS, flag for user review (do not auto-fail)" path. That is the goalpost-softening the original spec §5.3 forbade in advance, dressed in Wilson-score notation. The spec's own §1 rationale is "the right response per the original spec's no-margin condition (§5.3) is recalibrate in place, not soften pass criteria" — and then it softens pass criteria.
  - **What kills the finding (if anything):** Either (a) raise the pilot to N where 1 cap-violation falls cleanly outside the broken threshold (N=15 makes the 1/15 = 6.7% point estimate clearly inside §2.8's >2/20 = 10% boundary), or (b) declare 1/5 = inconclusive, re-author, and re-pilot. There is no admissible path through "borderline-PASS at N=5."
  - **What you'd do:** Strike "1/5 = borderline PASS" from §5 of the recalibration spec. Replace with "1/5 = inconclusive, raise N to 15 for that scenario." Cost increment: ~10 extra paired trials × ~$0.30 each = $3 over the existing N=5 budget.

- **[high]** The N=5 Sonnet pilot will run on a recalibrated scenario corpus while the failing pilot was run on the unrecalibrated corpus and a different ToT model. The result is uninterpretable.
  - **Where:** Recalibration spec §5: "Configuration: ... Sonnet 4.6 ToT (per ADR-0006 Amendment 1; claude-sonnet-4-6). Wall budget: ~30 min. Cost budget: ~$5–7." Morning pilot (`LESSONS.md:41-49`) ran Pro (Gemini 2.5). ADR-0006 Amendment 1 (line 125+) ratifies the Pro→Sonnet swap. No before/after on the same corpus exists.
  - **Why it matters:** Two confounded variables changed at once: scenario grids and ToT model. If the N=5 passes, the Δ test cannot distinguish "recalibration worked" from "Sonnet sees the cliff differently than Pro." If it fails, the same ambiguity in the other direction. The minimum honest pilot design is 2×2 (Pro/Sonnet × old/new grids), not 1 cell.
  - **What kills the finding (if anything):** A Pro run on the new grids OR a Sonnet run on the old grids — either single cell breaks the confound. Cost: ~$1.50 for one of them.
  - **What you'd do:** Before the full N=5 Sonnet × new-grids run, do N=2 Pro × new-grids on a UTC day with quota headroom. If `t_predicts` and Bot in-window rate both move toward target, the recalibration carried the load and the model swap is incidental.
